# photonics-ml-pipeline documentation

## Overview

This project demonstrates a complete, reproducible research pipeline that turns a piece
of physics into verified numerics, machine-learning features, a trained model, and
portable generated C -- with benchmarks and auto-generated API docs.

## The pipeline

```
Physics -> SymPy -> Numerical verification -> Feature extraction
        -> Machine learning -> Generated C -> Benchmarking -> Documentation
```

1. **Physics / SymPy** (`physics/`) -- The Gaussian-beam relations are derived
   symbolically. `SymbolicExpression` provides `gradient`, `hessian`, `cse`, and
   `lambdify`. Equation: `w(z) = w0 sqrt(1 + (z/z_R)^2)`, `z_R = pi w0^2 / lambda`.
2. **Numerical verification** (`physics/gaussian_beam.py`, `optics/abcd.py`,
   `photonics/dispersion.py`) -- Independent formulations (ABCD `q`-parameter, all-pass
   dispersion) cross-check the symbolic model. Dispersion is unitary (energy conserving).
3. **Feature extraction** (`feature_extraction/`) -- Eight interpretable features
   (energy, centroid, RMS width, peak, kurtosis, spectral centroid/bandwidth, Tamura
   contrast) summarize a field or image.
4. **Machine learning** (`ml/`) -- A PyTorch `Dataset` builds standardized feature
   vectors; a small MLP classifies beam-divergence class with high accuracy.
5. **Generated C** (`c_codegen/`) -- The finalized symbolic expression is emitted as
   optimized C (with common-subexpression elimination), plus Fortran and JavaScript. C
   output is written to `src/`/`include/` and verified against `lambdify`.
6. **Benchmarking** (`scripts/benchmark.py`) -- Times SymPy, NumPy, Torch, and compiled C.
7. **Documentation** (`scripts/gen_docs.py`) -- Regenerates `api.md` from docstrings.

## Assumptions and limitations

- Paraxial, monochromatic optics; spatial units are micrometers.
- Synthetic ML data; classes separable by beam-width statistics.
- Second-order dispersion only (single parameter `D`).
