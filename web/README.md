# `web/` — Static viewer for GS phase recovery

A zero-backend HTML/JS viewer for the dispersion-assisted Gerchberg-Saxton
phase-recovery work in this repo. It also renders the 4-port intensities of
the 90&deg; optical hybrid described in `Yiming_Hybrid90deg.mlx`.

## What is in here

```
web/
├── index.html         entry point.  Strict CSP, SRI on CDN scripts
├── js/app.js          ES-module: data load, Plotly 2D, VTK.js 3D surface
└── data/sample.json   precomputed GS iterates + hybrid outputs
```

## Local preview

```sh
# any static server works.  python is convenient because it ships with macOS / Linux:
python -m http.server -d web 8000
# then open http://localhost:8000
```

`file://` URLs work too but some browsers (Chrome) reject the JSON `fetch` over
that scheme. Use a real server.

## Refresh the dataset

`data/sample.json` is regenerated by the build script in the notebook
`jalali_phase_retrieval_and_feasibility.ipynb` (Part A). Run the GS cells and
pipe the snapshots into the same JSON schema; the viewer will pick it up on
reload.

## Security

See [`/SECURITY.md`](../SECURITY.md). TL;DR: strict CSP, SRI, no inline JS,
referrer disabled, CodeQL workflow under `.github/workflows/codeql.yml`.

Before deploying, run `./tools/update-sri.sh` to populate the
`integrity="sha384-INTEGRITY-TODO-…"` placeholders. The page will refuse to
load CDN scripts until you do.

## GitHub Pages

To serve this directly with Pages, in **Settings → Pages** pick **Deploy from a
branch → main → `/docs`**, or move `web/*` into `docs/`. Pages adds reasonable
default response headers, but it does not respect a per-repo response-header
config, which is why we put the CSP in a `<meta http-equiv>` tag inside
`index.html`.
