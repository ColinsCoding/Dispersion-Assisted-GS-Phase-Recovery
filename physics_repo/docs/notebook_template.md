# Notebook conventions

Every chapter notebook is generated by a script `tools/build_NN_<slug>.py` that uses `tools/nbkit.py`,
so structure is uniform and regeneration is reproducible (`make notebooks`).

## Required sections, in order

1. **Front matter** -- title and the six-question table (physics, mathematics, algorithm, software,
   experiment, engineering application).
2. **Setup** -- `nbkit.setup_cell()`: loads NumPy, Pandas, SymPy, Matplotlib, `physkit`, and an
   optional PyTorch via `physkit.optional_torch()`.
3. **English explanation** -- prose, subject-verb-object sentences.
4. **Mathematical derivation** -- the equations, from first principles.
5. **Dimensions and SI units** -- a `physkit.units` check that the result carries the right dimension.
6. **SymPy derivation** -- the symbolic reference; results are asserted.
7. **NumPy implementation** -- the numerical computation.
8. **Pandas tables** -- tabulated results.
9. **Matplotlib plots** -- figures in the repository style (`physkit.plotting.use_style`).
10. **PyTorch (optional)** -- guarded by `if torch is not None:`; NumPy stays authoritative.
11. **Exercises** -- at least three.
12. **Engineering applications** -- the computer-engineering connection, ending in a
    subject-verb-object summary.

## Rules

- Reuse earlier code through `physkit`; do not redefine constants, units, or linear-algebra helpers.
- Assert every claimed identity (SymPy `== 0`, NumPy `np.allclose`) so execution is a self-test.
- No emojis, no motivational language, no filler. Optimize for correctness, readability, and
  reproducibility.
- Prefer object-oriented design where it clarifies (e.g. `Dimension`, `Quantity`).

## Computer-engineering thread

Each chapter closes one link of the chain:
Hamiltonian -> matrix -> vector -> detector -> ADC -> firmware -> DSP -> CUDA -> visualization ->
scientific interpretation. The capstone (28) assembles the whole chain into one instrument pipeline.
